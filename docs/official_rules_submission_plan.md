# Official Rules Submission Plan

This project is built for the Microsoft Agents League Contest @ AI Skills Fest, Reasoning Agents track.

## Required submission assets

The official rules require the project submission to include:

1. A project description explaining:
   - Features
   - Functionality
   - Problem solved
   - Technologies used

2. A demo video:
   - Maximum 5 minutes
   - Uploaded to YouTube or Vimeo
   - Must not include third-party copyrighted/trademarked material unless permission is obtained

3. A public GitHub repository:
   - Source code for the Agent
   - No secrets, no API keys, no credentials
   - Clear README

4. An architecture diagram showing how the solution uses:
   - Microsoft Foundry and/or
   - Microsoft 365 Copilot and/or
   - GitHub Copilot

5. Team member information if applicable:
   - Include Microsoft Learn usernames for all participants if submitting as a team

## Our submission positioning

Project: CertPilot AI — Multi-Agent Certification Readiness Coach  
Track: Reasoning Agents  
Core technology story: Microsoft Foundry-style multi-agent workflow with Foundry IQ-style grounding, Work IQ-inspired context, and Fabric IQ-inspired semantic modeling.

## Judging alignment

### Accuracy & Relevance
CertPilot AI directly matches the Reasoning Agents scenario by managing enterprise certification readiness through role-based learning paths, study planning, assessments, and manager insights.

### Reasoning & Multi-step Thinking
The system decomposes the learner request into specialized agent tasks:
- Learning Path Curator
- Study Plan Generator
- Engagement Agent
- Assessment Agent
- Manager Insights Agent

### Creativity & Originality
The project turns certification preparation into an enterprise readiness intelligence system, not just a chatbot.

### User Experience & Presentation
Streamlit UI provides a clear demo flow, agent outputs, readiness score, sources, and manager recommendations.

### Reliability & Safety
The project uses synthetic data only, excludes PII, avoids credentials, exposes grounding sources, uses interpretable scoring rules, and documents responsible AI limitations.
